Pagination

A control to navigate through multiple pages when filtering search results.

Design so you don’t need pagination

Pagination hides content from the user. It can display a lot of unhelpful information that gets in the way to what the users are looking for. This may indicate the design isn’t working.

Interaction principles

You can avoid pagination by:

  • setting the default number of results per page (try 50 items per page, although do test with users to get it right for that service)
  • avoid infinite scrolling as this is problematic for keyboard-only users

Content principles

Consider what the sorting order should be to make the search more relevant. The more specific it is to the user's search criteria, the better the search results.

Services using non-paginated search results

  • Returns and movement
  • Passport application search

If you do require pagination, ensure that:

  • there is good colour contrast
  • font size is legible
  • large clickable hit areas are provided and are clearly spaced out
  • current page is clearly identified
  • set a maximum list of 5 pages
  • provide 'previous' and 'next' links, don't rely on icons alone
