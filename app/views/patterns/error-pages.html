Error pages

Inform a user when something goes wrong.

When a user visits a web page, the server responds with a HTTP status code. In most cases the user journey isn't affected - unless there is an error. Error statuses are sorted into two groups - 400 to 499 are client-side errors and 500 to 599 are server-side errors.

Error pages should:

  • be avoided - do the hard work to make sure they don't happen
  • use plain English and avoid technical jargon
  • avoid listing the error code - a user shouldn't need to know the error code
  • make it clear how to get help
  • if possible tell the user if they can complete the transaction on another channel

404 page

404 errors (sometimes known as 'page not found errors') tell a user the page they were trying to access is not there. Users encounter 404 errors when:

  • we make a mistake - for example a user follows a link that is broken
  • the user makes a mistake - inputting a link incorrectly

500 page

500 errors happen when something goes wrong on the server-side. Common causes are when the server is down or maintenance work is happening.

You do not need to show different error pages for different server problems. 500 pages should only be used for short periods of time. If there is going to be a disruption to users for an extended period use shutter pages like these HMRC examples.
